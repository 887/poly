## Client Trait (`ClientBackend`)

All messenger backends implement `poly_client::ClientBackend` in `clients/client/src/lib.rs`. The trait provides a uniform async API:

- Authentication / session management
- Server and channel listing
- Message send/receive
- User profiles, friends, presence
- Real-time event streaming

Since Phase 2.14, all backend clients are compiled as **WASM Component Model binaries** (`.wasm`) and loaded by the plugin host at runtime. See [1.1-wasm-plugin-system.md](1.1-wasm-plugin-system.md).

## Feature Flags

Each backend is compile-time gated. In practice, feature flags now control whether a `.wasm` blob is embedded in the binary (`embed-demo`, `embed-stoat`, etc.) rather than whether a Rust crate is linked.

```toml
[features]
embed-demo = []
embed-stoat = []
dev-plugins = []     # gate Discord/Teams until they're production-ready
```

## URL Routing

Every account-scoped URL carries `/:backend/:account_id/`:

```
/:backend/:account_id/dms
/:backend/:account_id/dms/:channel_id
/:backend/:account_id/friends
/:backend/:account_id/channels/:server_id
/:backend/:account_id/channels/:server_id/:channel_id
/:backend/:account_id/settings
```

`BackendType::from_slug()` maps the URL segment to the backend enum.

## UI Dispatch

Backend-specific UI is selected at render time via a `match` on `BackendType`:

```rust
pub fn backend_server_context_menu_items(backend: BackendType, ...) -> Element {
    match backend {
        BackendType::Demo => demo::server_context_menu_extras(...),
        BackendType::Stoat => stoat::server_context_menu_extras(...),
        // ...
    }
}
```

Per-backend UI modules live in `crates/core/src/ui/account/{backend}/` and are feature-gated with `#[cfg(feature = "...")]`.

## Data Flow

```
UI Layer (Dioxus signals)
    ↓ Signal<ChatData> + Signal<AppState>
ClientManager (account_id → BackendHandle)
    ↓ (one per active account)
Plugin Host (wasmtime)
    ↓
WASM Plugin (poly-demo.wasm, poly-stoat.wasm, ...)
```

## Adding a New Backend

1. Create `clients/{name}/` crate, implement `ClientBackend` WIT exports
2. Add `BackendType::{Name}` variant to `clients/client/src/types.rs`
3. Create `crates/core/src/ui/account/{name}/` UI module
4. Add dispatch arms + feature gate to `ui/account/mod.rs`
5. Add `"name"` to `BackendType::from_slug()` / `slug()`
