# 1.2 Host Bridge — DONE

> Status: DONE
> Sources: docs/plans/phase-2.21-host-bridge-unification-plan.md, CLAUDE.md
> Last updated: 2026-04-12

## Overview

Every shell mounts a `/host/*` route set on the **same port as its WASM bundle**. The three UI crates are Dioxus **fullstack** apps: `dx serve --fullstack` builds the WASM client and a native axum server from the same `src/main.rs`. The server half merges `poly_host::router(state)` into the Dioxus router before binding.

## Port Table

| Shell | Fullstack port | Storage backend |
|---|---|---|
| `apps/web` | 3000 | `storage.sqlite3` in OS data dir |
| `apps/desktop-electron` | 3001 | Same file |
| `apps/desktop` (Wry) | 3002 | Same file |
| `apps/poly-host` (standalone daemon) | 9333 | Same file |

All shells open the **same SQLite file** — accounts added in one shell are visible in all others.

## Data Directory

| Platform | Path |
|---|---|
| Linux | `$XDG_DATA_HOME/poly/storage.sqlite3` → `~/.local/share/poly/storage.sqlite3` |
| macOS | `~/Library/Application Support/poly/storage.sqlite3` |
| Windows | `%APPDATA%\poly\storage.sqlite3` |

Override with `POLY_DATA_DIR=/some/path`.

## Routes

```
GET  /host/status              — liveness ping
POST /host/kv/get              — app KV get
POST /host/kv/set              — app KV set
POST /host/kv/delete           — app KV delete
POST /host/kv/clear            — app KV clear_all
POST /host/exec                — spawn subprocess
POST /host/http                — one-shot HTTP via native reqwest
POST /host                     — LEGACY tagged-union dispatch (kept one cycle)
```

## Storage Implementations

The `crates/core/src/storage/` module selects an implementation at compile time:

| Impl | Config | Notes |
|---|---|---|
| `native` | Desktop/Electron fullstack | SQLite on disk via `rusqlite` |
| `web` | Browser WASM | IndexedDB (fallback for non-fullstack use) |
| `host-bridge` | WASM inside fullstack shell | Routes all calls through `/host/kv/*` |

The `host-bridge` implementation is what the WASM client bundle uses when running inside the fullstack server — it crosses the WASM↔native boundary via HTTP to the same-process server.

## Key Files

- `apps/poly-host/src/` — standalone daemon + shared `HostState`, `router()`, `resolve_data_dir()`
- `crates/host-bridge/src/` — HTTP client used by WASM-side code
- `crates/core/src/storage/host_bridge.rs` — storage impl that calls through the bridge
- `crates/core/src/storage/mod.rs` — cfg-gated storage dispatch

## Running `apps/web` with Fullstack

```bash
cd apps/web
dx serve --platform web --fullstack \
  @client --no-default-features --features "dev-plugins,web" \
  @server --platform server --no-default-features --features "dev-plugins,server"
```

`@server --platform server` is required — without it dx tries to build the server for `wasm32-unknown-unknown`.
