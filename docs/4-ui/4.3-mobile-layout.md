# 4.3 Mobile Layout — IN-PROGRESS

> Status: IN-PROGRESS
> Sources: docs/overall-plan.md, docs/archive/phases/
> Last updated: 2026-04-12

## Overview

Mobile layout collapses the three-pane desktop layout into swipeable pages.

## Three-Page Model

| Swipe | Content |
|---|---|
| Swipe left (or tap back) | Server list + channel list |
| Center (default) | Active chat / forum view |
| Swipe right | Member list for current channel |

## Responsive Breakpoints

Desktop (≥ 768px): all three panes visible simultaneously.
Mobile (< 768px): single-pane with gesture navigation.

## Current Status

Desktop layout is fully implemented. Mobile-specific CSS and gesture handling are planned but not yet implemented. Dioxus 0.7.3 supports iOS and Android targets; the mobile layout work is pending.

## Considerations

- Bottom navigation bar for mobile (vs. sidebar on desktop)
- Keyboard avoiding — chat input should scroll up when keyboard appears
- Safe area insets (iOS notch, Android gesture bar)
- Touch-friendly tap targets (minimum 44×44px)
