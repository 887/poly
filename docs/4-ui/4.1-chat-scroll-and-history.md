## Overview

Poly's chat view uses a **column-reverse CSS layout** for the message list.

- `scrollTop = 0` → visual bottom (newest messages)
- `scrollTop < 0` → scrolled up (older messages)
- `dist_from_bottom = (-scrollTop).max(0)`
- `dist_from_top = (scrollHeight - clientHeight - dist_from_bottom).max(0)`

The browser naturally preserves viewport position when content is prepended (older messages loaded above), so no manual scroll correction is needed for prepend operations.

## Key Implementation Details

### Infinite Scroll Up

`ChatHistoryUiState` has a `messages_loaded: bool` field that prevents a race condition where the history effect ran with empty messages on first render, set `has_more_before = false`, and then never retried.

```rust
pub struct ChatHistoryUiState {
    pub messages_loaded: bool,  // guards against race-init
    pub has_more_before: bool,
    pub channel_id: Option<String>,
}
```

The effect only returns early if `channel_id` matches AND `messages_loaded` is true.

### Scroll Position Memory

Per-channel scroll position is preserved when switching channels and restored on return.

### New Message Indicator

A "new messages" pill appears when scrolled up and new messages arrive. Clicking it scrolls to bottom.

### Anchor to Bottom

When the user is near the bottom (dist_from_bottom < threshold), new messages auto-scroll. When scrolled up, new messages do not force a scroll.
