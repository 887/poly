## Framework

**Dioxus 0.7.3** вҖ” reactive UI with RSX (Rust-based JSX), signals-based state management.

- `Signal<T>` вҖ” reactive state, auto-rerenders on change
- `use_context()` / `provide_context()` вҖ” pass state down the tree
- `#[component]` macro вҖ” define components as plain Rust functions
- RSX вҖ” `rsx! { div { class: "...", "text" } }` вҖ” Rust-embedded HTML-like syntax

## Dispatch Pattern

Backend-specific UI selected at render time via `match` on `BackendType`. Each backend has a per-module directory under `ui/account/` gated with `#[cfg(feature = "...")]`.

What goes in `common/` vs. per-backend:
- **`common/`** вҖ” message rendering, voice grid, emoji picker, account bar, notifications, friends
- **Per-backend** вҖ” server context menus, server settings, channel decorations, user profile cards, login flows

## Theming

CSS custom properties (`--var-name`) for all colors. Theme switching by swapping a CSS class. Themes in `crates/core/assets/styling/themes/`:
- `neutral-dark.css` вҖ” default dark theme
- `purple.css` вҖ” Discord-inspired
- `red.css` вҖ” Stoat-inspired

Dark mode is the default; light mode optionally follows device settings.

## i18n

`t!("key")` / `t!("key", arg: value)` macro. `.ftl` files in `locales/`. `use_i18n()` Dioxus hook for reactive locale switching.
