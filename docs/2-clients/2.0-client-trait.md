## Overview

`poly-client` (`clients/client/`) defines the shared trait and type system all messenger backends implement. It is the only crate that all backend plugin crates depend on.

**Crate:** `clients/client/` (`poly-client`)

## `ClientBackend` Trait

Located at `clients/client/src/lib.rs`. Key methods:

| Method | Description |
|---|---|
| `authenticate(credentials)` | Login with email/password, token, or guest |
| `logout()` | Invalidate session |
| `is_authenticated()` | Check session validity |
| `get_servers()` | List joined servers / communities |
| `get_channels(server_id)` | List channels in a server |
| `get_messages(channel_id, query)` | Fetch messages with pagination |
| `send_message(channel_id, content)` | Send a message |
| `get_user(id)` | Fetch user profile |
| `get_friends()` | List contacts / friends |
| `get_dm_channels()` | List direct message conversations |
| `get_groups()` | List group DMs |
| `get_notifications()` | Fetch notification feed |
| `get_voice_participants(channel_id)` | Voice channel member list |
| `get_presence()` / `set_presence()` | Online status |
| `event_stream()` | Real-time event stream (via `emit-event` in WASM) |
| `backend_type()` | Returns `BackendType` enum variant |
| `backend_name()` | Human-readable backend name |

Methods that a backend doesn't support return `ClientError::NotSupported`.

## Key Types (`clients/client/src/types.rs`)

| Type | Description |
|---|---|
| `BackendType` | Enum of all backends — `Demo`, `Stoat`, `Matrix`, `Discord`, `Teams`, `PolyServer`, `HackerNews`, `Lemmy`, `GitHub` |
| `Server` | A server / community / workspace |
| `Channel` | A channel within a server; `ChannelType`: `Text`, `Voice`, `Video`, `Forum` |
| `Message` | A chat message or forum post |
| `User` | A user profile |
| `Session` | An authenticated session |
| `AuthCredentials` | `EmailPassword`, `Token`, `Guest` |
| `MessageQuery` | Pagination params — `limit`, `before`, `after` |

## WIT Correspondence

The WIT interface (`wit/messenger-plugin.wit`) mirrors this trait exactly. All types in `poly-client` have a 1:1 WIT counterpart. The WIT file is the canonical interface; `poly-client` types must match.
