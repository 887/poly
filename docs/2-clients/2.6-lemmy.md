## Overview

Lemmy is a federated link aggregator / forum (ActivityPub). The Poly client supports subscribed communities as servers, forum-style posts with threaded comments, voting, and private messages. Multi-instance accounts are supported.

**Crate:** `clients/lemmy/` (`poly-lemmy`)
**API:** REST at `https://{instance}/api/v3/`

## Data Model Mapping

| Lemmy Concept | Poly Concept |
|---|---|
| Instance (`lemmy.ml`) | Account instance (like Matrix homeserver) |
| Subscribed community | Server |
| Community feed | Single forum channel per server |
| Post | Forum post (`ChannelType::Forum`) |
| Comment thread | Threaded messages under a post |
| Private message | DM channel |

## Key API Endpoints

| Operation | Endpoint |
|---|---|
| Login | `POST /api/v3/user/login` → JWT |
| Current user | `GET /api/v3/site` |
| Communities | `GET /api/v3/community/list` |
| Posts | `GET /api/v3/post/list?community_id=X` |
| Comments | `GET /api/v3/comment/list?post_id=X` |
| Private messages | `GET /api/v3/private_message/list` |
| User | `GET /api/v3/user?person_id=X` |

**Auth:** `Authorization: Bearer {jwt}`

## Authentication

JWT-based. Login with `username_or_email` + `password`. Token sent on all subsequent requests.

## Test Server

`servers/test-lemmy/` — Axum stub implementing Lemmy REST API v3. In-memory state, no real federation.

## Forum Channel Type

Lemmy uses `ChannelType::Forum` — see [4.2-forum-channels.md](../4-ui/4.2-forum-channels.md) for the UI layer that renders forum posts vs. chat messages.
