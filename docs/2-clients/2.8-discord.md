## Overview

Discord client for Poly. Significant challenges due to Discord's TOS and anti-automation measures.

**Crate:** `clients/discord/` (`poly-discord`) — stub
**Feature flag:** `discord` / `dev-plugins`

## Approach Decision (TBD)

Options under consideration:

1. **Unofficial user token API** — direct REST + Gateway WebSocket with user token. Works technically, but violates Discord TOS and risks account bans.
2. **Official Bot API** — requires a registered bot with server permissions; users must add the bot to their servers. Doesn't support DMs to arbitrary users.
3. **Webview approach** — embed a webview running discord.com, intercept API calls, extract tokens. Most TOS-risky but most complete feature coverage.
4. **Self-hosted Discord-compatible instance** — serve only users with their own Fosscord/similar instance.

**Decision deferred** to Phase 3.3. The `dev-plugins` feature flag gates Discord in the UI until a decision is made.
