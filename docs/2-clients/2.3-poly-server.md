## Overview

Poly Server is the project's own self-hosted chat backend. The full protocol spec is at [7.0-poly-server-protocol.md](../7-infrastructure/7.0-poly-server-protocol.md).

**Client crate:** `clients/server-client/` (`poly-server-client`)
**Server crate:** `servers/server/` (`poly-server`)
**WASM plugin:** yes

## Authentication

Ed25519 challenge-response:

1. `POST /auth/accounts` — check existing accounts for this public key
2. `POST /auth/signup` — create account (email + public key + username)
3. `POST /auth/challenge` — get server challenge
4. `POST /auth/verify` — sign challenge, get JWT token

Multiple accounts per identity key are supported. `user_id` is required in challenge/verify when multiple accounts exist.

## Key Features

- JWT tokens, 30-day lifetime (configurable via `POLY_SERVER_JWT_EXPIRY_SECS`)
- Device revocation — each session tracked, `device.revoked` checked on every request
- REST API for all data + WebSocket at `/ws` for real-time events
- Attachment serving at `/attachments/:id` with access-control
