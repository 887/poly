## Overview

Git forge backend for **Forgejo**, **Gitea**, and **Codeberg** instances.
Surfaces repositories, issues, pull requests, and source code as Poly channels
via the Forgejo/Gitea REST API v1 (`/api/v1/`).

Unlike the GitHub client (which wraps the `gh` CLI), this client uses direct
HTTP via `poly_host_bridge::http::HttpClient` — no external tools required.

**Crate:** `clients/forgejo/` (`poly-forgejo`)

## Data Model

| Forge Concept | Poly Concept |
|---|---|
| Repository | Server |
| Issue list | Forum channel (`ChannelType::Forum`) |
| PR list | Forum channel |
| Source tree | Code channel (`ChannelType::Code`) — file browser + reader |
| Issue/PR comments | Threaded messages under a post |

## Channel Types

Each repo exposes three channels (same structure as the GitHub client):

- **Issues** (`fj-issues-{owner}-{repo}`) — forum-style issue list
- **Pull Requests** (`fj-pulls-{owner}-{repo}`) — forum-style PR list
- **Code** (`fj-code-{owner}-{repo}`) — source explorer with `list_files()` and `read_file()`

## Authentication

Personal access token (PAT). Generate at `Settings → Applications` on any
Forgejo/Gitea instance. The token is sent as `Authorization: token {token}`
on every API request.

## Supported Instances

- **Codeberg** (codeberg.org) — first-class shortcut via `ForgejoClient::codeberg()`
- Any self-hosted **Forgejo** instance
- Any self-hosted **Gitea** instance (API-compatible)

## Key Features

- Repository browsing (owner + collaborator repos, filtered by 2-year activity window)
- Issue and PR listing with state, comment counts, and timestamps
- Comment threads as threaded messages
- Source code file browser and reader (`ChannelType::Code`)
- Read-only initially; write operations planned for future iteration
- Private/archived repo filtering

## API Endpoints Used

| Endpoint | Purpose |
|---|---|
| `GET /api/v1/user` | Auth check + user profile |
| `GET /api/v1/user/repos` | List user's repos |
| `GET /api/v1/repos/{owner}/{repo}/issues` | List issues or PRs |
| `GET /api/v1/repos/{owner}/{repo}/issues/{number}/comments` | Issue comment thread |
| `GET /api/v1/repos/{owner}/{repo}/contents/{path}` | Source tree / file content |

## Feature Flag

`forgejo` — compiled in via feature flag in `crates/core/Cargo.toml`.
Enabled by default in all three app shells (web, desktop, electron).
