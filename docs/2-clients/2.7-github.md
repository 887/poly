## Overview

GitHub backend surfacing Issues, Pull Requests, and notifications as a Poly messenger channel. Uses the GitHub REST API via the `gh` CLI or direct API calls.

**Crate:** `clients/github/` (`poly-github`)

## Data Model

| GitHub Concept | Poly Concept |
|---|---|
| Repository | Server |
| Issue list | Forum channel (`ChannelType::Forum`) |
| PR list | Forum channel |
| Notifications | Notification feed |
| Issue comments | Threaded messages under a post |

## Authentication

GitHub personal access token or `gh auth token` (delegates to the `gh` CLI for token retrieval).

## Key Features

- Issue and PR browsing per repository
- Comment threads as threaded messages
- Notification feed (mentions, reviews, CI, etc.)
- Read-only; no write operations in initial implementation

`github` — compiled in via feature flag.
