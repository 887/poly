## Overview

The Demo client is a mock backend with hardcoded data used for UI development and testing without any network connections. It is always embedded in the binary and always available.

**Crate:** `clients/demo/` (`poly-demo`)
**WASM plugin:** yes (`wasm32-wasip2`)

## What It Provides

- Multiple demo servers with categories and channels (text, voice, video, forum)
- Several demo accounts (Cat, Dog, etc. with avatar images in `clients/demo/assets/`)
- Demo DM conversations
- Demo friend list
- Demo notifications
- Demo forum channel (`demo_forum`) with posts and threaded comments
- Randomized message content with timestamps
- Stubbed voice participants

## Animals / Avatars

Assets in `clients/demo/assets/`:
- `cat.png` — calico cat with pearl necklace, purple background
- `dog.png` — golden retriever with sunglasses, blue background
- `beaver.svg`, `hedgehog.svg` — added in Phase 3

## Session Slug

`demo` and `demo_forum` — separate slugs for regular demo and forum demo account.

## Feature Flag

Built-in, always compiled. No feature flag gate. The `embed-demo` build flag controls whether the `.wasm` is embedded at compile time.
