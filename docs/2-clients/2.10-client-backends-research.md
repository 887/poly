## Summary

Research compiled 2026-02-28. Key findings:

### Stoat (Revolt)

- No viable Rust SDK (`revolt-rs` 0.0.3 and `rive` both unmaintained 2+ years)
- Must build custom REST + WebSocket client from API docs at `developers.stoat.chat`
- Protocol: REST API + Bonfire WebSocket + Vortex WebRTC (voice)
- Self-hosted: yes, same API structure at different base URLs
- Auth: `POST /auth/session/login` → `x-session-token` header

### Matrix

- Official Rust SDK: `matrix-sdk = "0.16.0"` — same SDK as Element X
- Spec: `spec.matrix.org`
- E2EE built-in, cross-signing, room verification
- Spaces provide Discord-like server grouping

### Discord

- No official Rust client SDK
- Third-party: `serenity` (bot-only), `twilight` (bot-only)
- User-token usage violates TOS
- No safe, fully-featured approach known at time of research
- Decision deferred to Phase 3.3

### Microsoft Teams

- Official SDK: none for Rust
- Reference: `ttyms` crate for auth/message patterns
- Graph API: well-documented, REST-based
- OAuth2 via Azure AD — complex initial auth flow
