## Overview

Read-only (optionally authenticated) Hacker News backend. Maps the HN Firebase API onto the `ClientBackend` trait using a forum model — stories as forum posts, threaded comments as replies.

**Crate:** `clients/hackernews/` (`poly-hackernews`)
**API:** `https://hacker-news.firebaseio.com/v0/` (free, no auth required for reads)

## Data Model

```
Server: "Hacker News" (id: "hn")
  Category: "Stories"
    Channel: Top     (id: hn-top)
    Channel: New     (id: hn-new)
    Channel: Best    (id: hn-best)
  Category: "Ask & Show"
    Channel: Ask HN  (id: hn-ask)
    Channel: Show HN (id: hn-show)
  Category: "Jobs"
    Channel: Jobs    (id: hn-jobs-ch)
```

Comment threads: pseudo-channel `hn-post-{item_id}`.

## Key API Endpoints

| Endpoint | Returns |
|---|---|
| `/v0/topstories.json` | `Vec<u64>` (up to 500 story IDs) |
| `/v0/newstories.json` | Same |
| `/v0/item/{id}.json` | Story, comment, job, or poll |
| `/v0/user/{username}.json` | User profile |
| `/v0/updates.json` | Recently changed items (for polling) |

## Authentication

Optional, cookie-based via `POST https://news.ycombinator.com/login`. Required only for voting and posting comments. Anonymous read works without any auth.

## Event Stream

No WebSocket — polling `/v0/updates.json` every 30 seconds.

## Caching

- Story ID lists: 2-minute TTL
- Story items: 5-minute TTL
- Comments: 10-minute TTL
- User profiles: 30-minute TTL
- Max 10 concurrent item fetches per batch

## Test Server

`servers/test-hackernews/` — Axum stub of Firebase HN API with hardcoded mock stories.
