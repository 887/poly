## Overview

Stoat (formerly Revolt) is a self-hostable Discord-like chat platform. The Poly client is a custom REST + WebSocket implementation — no viable Rust SDK exists.

**Crate:** `clients/stoat/` (`poly-stoat`)
**WASM plugin:** yes (`wasm32-wasip2`)

## What's Working

- Authentication (`POST /auth/session/login` → session token)
- Server listing (`GET /servers`, `GET /users/@me/servers`)
- Channel listing (text, voice, DM, group DM)
- Message fetch and send
- User profile
- DM channels
- Live API tested against test server and real stoat.chat
- `test_signin` bypass for test server (no password needed via `/test/auth/token`)

## API Summary

| Operation | Endpoint |
|---|---|
| Login | `POST /auth/session/login` |
| Logout | `DELETE /auth/session/logout` |
| My servers | `GET /users/@me/servers` |
| Server info | `GET /servers/{id}` |
| Channel info | `GET /channels/{id}` |
| Messages | `GET /channels/{id}/messages` |
| Send message | `POST /channels/{id}/messages` |
| User | `GET /users/{id}` |

**Auth:** `x-session-token` header

**WebSocket (Bonfire):** `wss://ws.stoat.chat` (configurable for self-hosted)

## Self-Hosted Support

All base URLs configurable — same API structure. User enters the instance URL when adding the account.
