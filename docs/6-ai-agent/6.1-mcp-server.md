## Overview

`poly-chat-mcp` — an MCP server that exposes every `ClientBackend` method as a tool. Runs embedded in the Poly app or standalone.

**Crate:** `mcp/chat-mcp/` (`poly-chat-mcp`)

## Architecture

```
Poly App (:3000 / :3001 / :3002)
  ├── ClientManager (shared backend pool)
  └── poly-chat-mcp (mounted as /mcp/* routes)
          ↓
poly-cli (connects via HTTP, discovers tools at runtime)
```

No `mcp.json` configuration needed for internal use — the MCP runs as HTTP routes inside the app. External MCP clients (Claude Code) can also connect via the same HTTP endpoint.

## Implemented Tools

| Tool | Description |
|---|---|
| `health` | Liveness check |
| `login` | Authenticate with a backend |
| `test_signin` | No-password test server login |
| `list_servers` | List servers for an account |
| `list_channels` | List channels in a server |
| `list_dms` | List DM conversations |
| `list_friends` | List contacts |
| `get_messages` | Fetch messages from a channel |
| `list_plugins` | List registered WASM plugins |

## `test_signin` Tool

- Input: `{backend, url, username}` — no password
- Guard: only works when `url` contains `localhost` or `127.0.0.1`
- Calls `/test/auth/token` → logs in with returned token
- Available for: stoat, matrix, lemmy, discord, teams test servers

## Verified End-to-End

```bash
poly-cli call login --backend stoat --url http://localhost:9101 --username stoat --password testpass123
poly-cli call list_servers --backend stoat
poly-cli call list_channels --backend stoat --server_id SRV001
poly-cli call list_dms --backend stoat
poly-cli call get_messages --backend stoat --channel_id CH001 --limit 3
poly-cli call test_signin --backend stoat --url http://localhost:9101 --username raccoon
```
