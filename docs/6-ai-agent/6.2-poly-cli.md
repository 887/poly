## Overview

`poly-cli` — a thin MCP client that connects to a running `poly-chat-mcp` server and dynamically exposes all tools as CLI subcommands.

**Key design:** poly-cli never compiles against backend crates. It discovers tools at runtime via `tools/list`. When the MCP adds a new tool, poly-cli gets it automatically without recompilation.

## Usage

```bash
# Point at web app (default)
poly-cli call <tool_name> [--key value ...]

# Point at specific shell
poly-cli --url http://localhost:3001 call list_servers --backend stoat

# List available tools
poly-cli tools
```

## Dynamic Discovery

1. Connect to `{url}/mcp`
2. Call `tools/list` → get all tool schemas
3. Parse input args against the discovered schema
4. Call `tools/call` with the tool name + args
5. Print result

## Targeting Multiple Shells

| Shell | Default URL |
|---|---|
| poly-web | `http://localhost:3000` |
| poly-electron | `http://localhost:3001` |
| poly-desktop | `http://localhost:3002` |
| standalone MCP | `http://localhost:3010` |
