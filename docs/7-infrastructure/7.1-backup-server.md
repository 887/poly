## Overview

`poly-backup-server` (`servers/backup-server/`) — a self-hosted encrypted settings sync server. Users run their own instance to back up app settings across devices.

**Philosophy:** The server stores nothing it can read. All data is client-side encrypted before upload. The server is a dumb authenticated blob store.

## What Gets Synced

- Logged-in account tokens / credentials (encrypted)
- App settings (themes, favorites, preferences)
- NOT: message history (stays local / on the respective messenger servers)

## Auth Model

No user accounts in the traditional sense. Auth is:

1. **Public key identity** — user's Ed25519 public key IS their user ID
2. **Optional server passphrase** — operator sets a passphrase; clients must know it to register
3. **Account limit** — operator sets max accounts; refuses new keys once limit reached
4. **PoW rate limiting** — clients solve a proof-of-work puzzle before uploading

The server has no way to identify who you are — only your public key hash.

## Encryption

All blobs are encrypted client-side before upload:
- Symmetric key derived from X25519 private key (HKDF)
- Encrypted with AES-256-GCM or XSalsa20-Poly1305
- Server stores: `{public_key_hash → encrypted_blob}`

Sync flow:
1. Encrypt settings blob locally
2. Push encrypted blob to server (identified by public key hash)
3. On another device: pull blob by public key hash, decrypt locally

## Multi-Server Support

Users can add multiple backup servers in settings. The app syncs to all configured servers.

## Web Interface

The backup server includes a Dioxus + Axum web interface for administration (viewing account count, managing limits, server status).

## Key Configuration

| Env Var | Description |
|---|---|
| `POLY_BACKUP_PASSPHRASE` | Optional passphrase required to register |
| `POLY_BACKUP_MAX_ACCOUNTS` | Max accounts (default: unlimited) |
| `POLY_BACKUP_DATA_DIR` | SurrealKV storage directory |
