## Summary

Poly Server (`servers/server/`) is a self-hosted chat backend exposing:
- REST API over HTTP/1.1 or HTTP/2 for all data operations
- WebSocket at `/ws` for real-time server-push events
- Attachment serving at `/attachments/:id` with access control

**Auth:** JWT (HS256), 30-day lifetime, device-tracked with revocation support.

## Authentication Flow

1. `POST /auth/accounts` — find existing accounts for a public key
2. `POST /auth/signup` — create account (email + Ed25519 public key + username)
3. `POST /auth/challenge` — get server-generated challenge
4. `POST /auth/verify` — sign challenge with Ed25519, receive JWT + device_id

Multiple accounts per identity key are supported. `user_id` required in challenge/verify when multiple accounts exist.

## Key Endpoints

| Category | Endpoints |
|---|---|
| Auth | `/auth/accounts`, `/auth/signup`, `/auth/challenge`, `/auth/verify` |
| User | `/users/me`, `/users/{id}` |
| Servers | `/servers`, `/servers/{id}`, `/servers/{id}/members` |
| Channels | `/channels/{id}`, `/channels/{id}/messages` |
| DMs | `/dms`, `/dms/{id}/messages` |
| Attachments | `/attachments/{id}` |
| WebSocket | `/ws` |

## Error Responses

```json
{ "error": "human-readable description" }
```

Standard HTTP status codes: 400 bad request, 401 unauthorized, 403 forbidden, 404 not found, 409 conflict, 500 internal.
