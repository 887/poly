## Overview

`apps/web` — Dioxus fullstack app targeting Chrome/Chromium. Port 3000.

## Running

```bash
cd apps/web
dx serve --platform web --fullstack \
  @client --no-default-features --features "dev-plugins,web" \
  @server --platform server --no-default-features --features "dev-plugins,server"
```

`@server --platform server` is required — without it dx tries to build the server for `wasm32-unknown-unknown`.

## Via MCP (Recommended)

```bash
cargo run --bin poly-web-devtools-mcp
```

Or VS Code task: `Serve: web (MCP + Chromium)`.

## Key Details

- **Port 3000** (never 8080, never with `--hotpatch`)
- **No `--hotpatch`** — Dioxus 0.7.3 WASM doesn't support experimental hotpatch
- CDP on port **9222** for devtools / MCP
- Auto-restarts Chromium if closed
- Cleanup script: `./scripts/web-cleanup.sh`

## Fullstack Features

- WASM client + axum server from one `src/main.rs`
- `/host/*` routes for native storage access
- Same `storage.sqlite3` as desktop shells

## Common Issues

| Problem | Fix |
|---|---|
| "Your app is being rebuilt" (stuck) | `pkill -f "dx serve"` then restart MCP |
| CDP connect fails | `pkill -f "remote-debugging-port=9222"`, restart MCP |
| Port 3000 in use | `pkill -f "dx serve.*3000"` |
