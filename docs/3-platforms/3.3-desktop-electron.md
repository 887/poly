## Overview

`apps/desktop-electron` + `apps/desktop-electron-web` — Electron thin shell wrapping the Dioxus WASM app. Port 3001.

## Architecture

- `apps/desktop-electron/` — Dioxus fullstack app (WASM client + axum server)
- `apps/desktop-electron-web/electron/main.js` — Electron main process that opens a `BrowserWindow` and loads from port 3001
- Custom frameless title bar in Electron

## Key Implementation Notes

### ELECTRON_RUN_AS_NODE

VS Code and Claude Code terminals set `ELECTRON_RUN_AS_NODE=1`. This causes Electron to run as plain Node.js where `require('electron')` fails. The MCPs strip this env var when spawning Electron processes.

### Frameless Window

Use `frame: false` only. Do NOT combine with `titleBarStyle: 'hidden'` or `titleBarOverlay: false` — these conflict on Linux and cause pixel offsets.

### CSS Layout

`.main-layout` uses `height: 100%` (not `100vh`) to respect the Electron custom titlebar (34px).

## Debug / MCP

- CDP port: **9224**
- MCP server: `mcp/electron-devtools-mcp/`
- MCP binary: `poly-electron-devtools-mcp`

## Orphan Cleanup

The Electron MCP kills stale processes by matching `poly-desktop-electron-web` in the command line (catches main, GPU, network, renderer processes). Also kills by `dx serve` port pattern.

## Running

```bash
# Start the fullstack app
cd apps/desktop-electron
dx serve --platform web --fullstack --port 3001

# The Electron shell (in another terminal or via MCP)
cd apps/desktop-electron-web
npm start
```
