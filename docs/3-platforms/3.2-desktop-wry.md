## Overview

`apps/desktop` + `apps/desktop-web` — Dioxus desktop app using Wry (WebKit2GTK on Linux, WebKit on macOS, WebView2 on Windows). Port 3002.

## Architecture

- `apps/desktop/` — Dioxus fullstack app (WASM client + axum server)
- `apps/desktop-web/` — Wry native shell that opens a window and loads from port 3002

## Key Implementation Notes

### Linux: `build_gtk()`

`wry::WebViewBuilder::build_gtk()` must receive `window.default_vbox()`, NOT `window.gtk_window()`. Using `gtk_window()` results in a 0x0 viewport.

### CSS Layout

`.main-layout` uses `height: 100%` (not `100vh`) to respect the flex parent's allocated size.

## Debug / MCP

- Eval-bridge port: **9223**
- MCP server: `mcp/desktop-devtools-mcp/`
- MCP binary: `poly-desktop-devtools-mcp`

## Running

```bash
# Start the fullstack app
cd apps/desktop
dx serve --platform web --fullstack --port 3002

# The Wry shell (in another terminal or launched by MCP)
cargo run -p poly-desktop-web
```

## Orphan Cleanup

The desktop MCP kills stale processes by matching `poly-desktop-web` in the command line. Also kills by `dx serve` port pattern.
